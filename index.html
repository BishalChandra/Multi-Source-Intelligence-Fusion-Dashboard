Logic 01
Data ingestion & validation
Every record entering the system must contain a valid latitude (−90 to 90), longitude (−180 to 180), a label string, and a confidence score (0–100). Invalid records are rejected with an error notification before they reach the map.
if (!lat || lat < -90 || lat > 90) → reject("Invalid latitude") if (!lon || lon < -180 || lon > 180) → reject("Invalid longitude") if (!label || label.trim() === "") → reject("Label required") if (conf < 0 || conf > 100) → reject("Confidence must be 0–100")
Logic 02
Node mapping & rendering
Each validated record is converted to a pixel position on the map canvas using a linear interpolation of its lat/lon against the visible map bounding box. Nodes are color-coded: blue (OSINT), amber (HUMINT), green (IMINT).
x = (lon - minLon) / (maxLon - minLon) * mapWidth y = (maxLat - lat) / (maxLat - minLat) * mapHeight color = { osint: "#3a7bd5", humint: "#e8a842", imint: "#3dae7a" }
Logic 03
Hover-and-view pop-up trigger
When the analyst's cursor enters a node element, a pop-up modal is positioned adjacent to the node and populated with its metadata. The pop-up is repositioned if it would overflow the map boundary. Mouse-leave hides the modal.
onMouseEnter(node) → populate popup → check overflow → show if (popupRight > mapWidth) → flip popup to left side of node if (popupTop < 0) → shift popup below node onMouseLeave → hide popup
Logic 04
Source layer filtering
Each source type (OSINT, HUMINT, IMINT) has an independent visibility toggle. When toggled, the render function filters the node array and redraws only visible nodes. Counts in the sidebar update accordingly.
visibleNodes = NODES.filter(n => srcVisibility[n.type] === true) renderNodes(visibleNodes) updateSidebarCounts(visibleNodes)
Logic 05
Confidence scoring & color grading
Confidence bars in the Intel Table are color-graded in three tiers: high confidence (≥80%) renders green, medium (60–79%) renders amber, low (<60%) renders red. This gives analysts an instant visual risk signal.
if (conf >= 80) → barColor = "#3dae7a" // green — high if (conf >= 60) → barColor = "#e8a842" // amber — medium if (conf < 60) → barColor = "#d05050" // red — low
Logic 06
Duplicate node detection
Before adding a new node, the system checks whether an existing node with the same label and nearly identical coordinates (within 0.01° tolerance) already exists. Duplicates are flagged with a warning rather than silently overwritten.
isDuplicate = NODES.some(n => n.label === newNode.label && Math.abs(n.lat - newNode.lat) < 0.01 && Math.abs(n.lon - newNode.lon) < 0.01 ) if (isDuplicate) → warn("Duplicate node detected") → abort
Logic 07
Activity feed & session logging
Every ingestion event is prepended to the live activity feed with a UTC timestamp and source badge. The bulk log tracks all records ingested during the session. A session reset restores the default node set without clearing the feed history.
onIngest(node) → feedLog.prepend({ time: utcNow(), type, label }) bulkLog.prepend({ label, conf, source }) statusBar.update(totalCount, lastLabel)
